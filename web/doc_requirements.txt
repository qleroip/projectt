ЧАСТЬ 1: Реализация жизненного цикла сущности
Требование
Выбрать одну основную сущность проекта и реализовать для неё:
1. Поле состояния
status: string (или enum)
2. Минимум 4 состояния
CREATEDACTIVECOMPLETEDCANCELLED
(названия можно адаптировать под проект)
3. Разрешённые переходы
Студент обязан явно задать:
CREATED -> ACTIVEACTIVE -> COMPLETEDACTIVE -> CANCELLED
4. Запрещённые переходы
Например:
COMPLETED -> ACTIVE CANCELLED -> ACTIVE
Логика
Система должна:
проверять текущий статус
валидировать переход
выбрасывать ошибку при нарушении
Ошибки
Ситуация
Код
не найдено
404
недопустимый переход
409
Критерии
есть enum/константы статусов
есть логика проверки переходов (НЕ в контроллере)
есть минимум 2 запрещённых перехода
ошибка 409 возвращается корректно
Часть 2: Работа со связями + транзакция
Требование
Реализовать операцию, которая:
Затрагивает минимум 2 связанные сущности
Пример
Создание основной сущности+ автоматическое создание связанной
Конкретные требования
1. Должна быть связь:
One-to-Many или Many-to-One
2. Операция должна:
создавать объект A
создавать связанные объекты B
сохранять всё в одной транзакции
Обязательное условие
Если ошибка при создании одного из элементов:
ничего не должно сохраниться
Критерии
есть связь в ORM (relationship)
операция выполняется через service
используется транзакция
при ошибке данные не сохраняются частично
